# 工具分层决策规则 / Tool Routing Rules

> 核心理念：选择哪个 AI 本身就是一个需要学习的认知决策。

---

## 决策规则（核心）

### 规则1：任务可独立验证吗？

- **是**（如：单句纠错、变位查询）→ 战术层
- **否**（如：需要多步推理、整合多源信息）→ 战略层

### 规则2：任务有客观答案吗？

- **是**（如：时态使用、动词变位）→ 战术层（更快）
- **否**（如：哪种表达更地道、文化语境判断）→ 战略层（更可靠）

### 规则3：对话长度预期如何？

- **1-3 轮**（如：纠错、查询）→ 战术层
- **5+ 轮**（如：诊断、规划）→ 战略层

### 规则4：需要 AI 持续记忆上下文吗？

- **否**（单点任务）→ 战术层
- **是**（如：多步对话、整篇文章分析）→ 战略层

### 规则5：紧迫程度如何？

- **高**（如：临场问题、当下作业）→ 战术层（更快）
- **低**（如：可以等几小时的规划）→ 战略层（更深入）

## 跨 AI 协作模式

### 模式1：战术+战略串联

```
用户问豆包（战术层）：这个句子对吗？
   ↓
发现问题
   ↓
用户问 CodeX/DeepSeek（战略层）：为什么错？背后的规则是什么？同类错误还有哪些？
   ↓
得到深度解释
   ↓
回到豆包（战术层）：快速验证其他类似句子
```

### 模式2：双 AI 对比验证

```
用户向豆包和 Codex 提出同一问题
   ↓
对比两个 AI 的回答
   ↓
一致 → 较高可信度
   ↓
不一致 → 独立查证（教材、老师、其他来源）
```

### 模式3：战略层主导 + 战术层辅助

```
Codex（战略层）做整体规划：制定学习路径
   ↓
豆包（战术层）执行具体任务：日常纠错、查询
   ↓
Codex（战略层）做阶段反思：评估进展、调整计划
```

## 不同 AI 平台的适配建议

如果你使用其他 AI 平台作为战术层或战略层，以下是适配建议：

### 战术层替代方案
- 任何响应快、对话自然的 AI：通义千问、文心一言、ChatGPT、Claude
- 关键指标：响应时间 < 3 秒，对话风格自然

### 战略层替代方案
- 任何长上下文强、推理能力强的 AI：Claude（推荐）、GPT-4、DeepSeek-V3+
- 关键指标：支持 32K+ token 上下文，多步推理不丢失

### 双 AI 选择的原则
1. 两个 AI 应有**明显的能力差异**（同质选择意义不大）
2. 战术层和战略层的 AI 应当是**互补关系**而非竞争关系
3. 两个 AI 的**使用界面应当易于切换**（减少切换成本）
